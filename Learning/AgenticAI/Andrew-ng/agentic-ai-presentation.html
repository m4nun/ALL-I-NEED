Module 1

What Is Agentic AI?

One-shot prompting is like asking someone to write an essay from first word to last without ever using backspace.

Humans don't write their best work that way — and neither do LLMs.

Agentic Workflow

  1. Write an essay outline
  2. Conduct web research
  3. Write the first draft
  4. Read and critique the draft
  5. Revise based on the critique
"An agentic AI workflow is a process where an LLM-based app executes multiple steps to complete a task."

Core skill: Decomposing complex tasks into smaller steps executed by an LLM, code, or a function call.

Spectrum

Degrees of Autonomy

Why "agentic"? To move past the binary debate of "true agent or not."

Less Autonomous

Deterministic

Hard-coded sequence by programmer. LLM only generates text. E.g., generate search terms → web search → write essay.

Semi-Autonomous

Tool Selection

LLM decides which tools to call, but the tool set is predefined.

Highly Autonomous

Self-Directed

LLM decides the sequence of steps, may write new functions or create tools.

Most valuable applications today sit on the less-to-semi-autonomous end.

Benefits

Why Agentic AI?

1

The improvement from one model generation to the next is dwarfed by wrapping the older model in an agentic workflow.

2

Generate search terms in parallel, fetch nine web pages simultaneously, then synthesize — much faster than a human.

3

Modularity

Swap search engines, LLMs, or tools independently — without rebuilding the whole system.

Framework

Task Decomposition

Always ask: "Can each step be done by an LLM, a short piece of code, a function call, or a tool?"

1. Direct Generation

One step. Good enough for simple tasks.

2. Decompose into 2-3 Steps

Outline → search → write. Fixes quality issues.

3. Further Decompose

First draft → reflect → revise. Fixes disjointed output.

Examples: Essay (outline → search → write) · Customer email (extract → query → draft) · Invoice (PDF code → LLM extract → API)

Module 2

Reflection

Reflection

The Reflection Pattern

Just as humans reflect on their drafts, LLMs can do the same.

  1. Generate v1 output
  2. Feed v1 back with a reflection prompt
  3. Generate improved v2
Email Writing

v1 draft → "Check tone, verify facts, check dates" → v2 improved

Code Generation

v1 code with bugs → reflect with reasoning model → v2 fixed code

Evidence

Why Not Just Direct Generation?

Research from Madaan et al. shows reflection consistently outperforms zero-shot prompting across many tasks and models.

Validate Outputs

Structured HTML/JSON outputs checked for correctness.

Check Completeness

"How to brew the perfect cup of tea" — did we miss steps?

Review Branding

Domain names checked for unintended meanings or pronunciation issues.

"The reflection design pattern isn't magic, but it can often give a modest bump in performance."
Power Multiplier

External Feedback

Pure prompt engineering plateaus. External feedback breaks through the ceiling.

Code Execution

Run code → feed error messages → LLM fixes bugs.

Competitor Detection

Regex match on output → feed back as criticism if competitor mentioned.

Fact-Checking

Web search to verify claims → feed snippets back to LLM.

Word Count

Count words → if over limit, feed count back → LLM tightens text.

"Reflection with external feedback is much more powerful than reflection using the LLM as the only source."
Evaluation

Quantify the Impact

Don't assume reflection helps — measure it.

  1. Collect 10-15 questions with ground-truth answers
  2. Run workflow without reflection
  3. Run with reflection
  4. Compare accuracy

Module 3

Tool Use

Giving LLMs capabilities beyond text

The LLM decides when to call a tool.

Mechanism

What Are Tools?

Tools are functions an LLM can request to call. The LLM autonomously decides whether to use them.

1. Description

Tool descriptions (name, description, parameters) are provided in the prompt.

2. Decision

The LLM decides whether to use a tool based on the query.

3. Execution

If the LLM requests a tool call, your code executes it and returns the result.

Examples: Web search for restaurant recommendations · Database query for retail · Calculation for interest rates · Code execution for general math
Modern Syntax

AISuite & JSON Schema

The modern approach uses libraries like AISuite to automatically generate JSON schemas from function docstrings and type annotations.

  • Schema includes name, description, parameters
  • Passed to LLM so it knows when and how to call
  • Library handles the function-calling loop
  • max_turns=5 prevents infinite loops
# Python example
response = client.chat.completions.create(
model="gpt-4o",
messages=messages,
tools=[getCurrentTime],
max_turns=5
)
Power Tool

Code Execution

Instead of creating a separate tool for every operation, let the LLM write and execute Python code.

  1. Prompt LLM to write code delimited with tags
  2. Extract code with regex
  3. Execute in a sandbox (Docker, E2B)
  4. Return output to LLM
  5. If execution fails, feed error back for debugging
Security Best Practice

Always run LLM-generated code in a sandbox.

Real incident: an agentic coder deleted Python files from a project directory.

Standard

MCP: Model Context Protocol

An Anthropic-proposed standard now widely adopted. Solves the N×M integration problem.

Without MCP

M applications × N tools = M×N custom wrappers

With MCP

M clients + N servers = M+N total work

Architecture: MCP Clients (apps) connect to MCP Servers (wrappers for Slack, GitHub, Google Drive, Postgres, etc.). Covers both data access (resources) and function calling (tools).

Practical Tips

Evaluations, Error Analysis & Optimization

The most important module.

Core Method

Eval-Driven Development

Build a quick, safe prototype. Then use evals to drive improvements.

01

Build Quick System

Extract four fields from invoices. Don't over-engineer.

02

Examine Outputs

Manually review 10-20 examples. Discover due dates are frequently wrong.

03

Create & Iterate

Annotate ground truth. Write eval code. Iterate prompts. Track accuracy.

"It's difficult to know in advance where the system will work. Build a quick system, look at it, and use that to prioritize."
Trace Analysis

Error Analysis & Prioritization

Agentic workflows have many components. Guessing which to improve is inefficient.

The Process

  1. Find the problem (e.g., "missing key points")
  2. Look at traces — intermediate outputs after each step
  3. Identify failing component
  4. Have a human expert evaluate each component
  5. Count subpar outputs per component
  6. Focus on the highest error count
Real Example

Research agent "missed key points." Traces revealed the web search engine returned too many blog articles instead of scientific sources.

Invoice Example

75% of errors came from SQL query generation — not PDF conversion or email drafting.

Iteration Speed

Component-Level Evaluations

End-to-end evals are expensive and noisy. Component-level evals let you iterate 10× faster.

  • Create gold-standard authoritative URLs
  • Measure overlap (F1 score)
  • Test search engines, result counts, date ranges
  • Validate with end-to-end eval once optimized
10×
Faster Iteration

Parallel team workflows: each team optimizes its own metric.

Fixes

How to Address Problems

Categorize fixes by component type.

Non-LLM Components
  • Tune hyperparameters (results, thresholds, chunk size)
  • Replace the component (swap search engines, RAG providers)
LLM-Based Components
  • Improve prompts (instructions, few-shot examples)
  • Try a different LLM
  • Decompose into simpler sub-steps
  • Fine-tune as last resort (expensive)
"I tend not to fine-tune a model until I've really exhausted the other options."
Optimization

Latency & Cost Optimization

Priority order: output quality → latency → cost

Benchmark Each Step

Time each component and calculate per-step cost. Identify bottlenecks.

Leverage Parallelism

Fetch multiple web pages simultaneously. Run independent LLM calls in parallel.

Cost Analysis

Calculate token costs and API fees per component. Focus on the most expensive ones.

"Getting output quality high is usually the hardest part. Only then focus on latency and cost."

Highly Autonomous Agents

Planning, Code Execution & Multi-Agent

Moving beyond hard-coded sequences.

Planning

Planning Workflows

The LLM generates a step-by-step plan to answer complex queries.

  • The sequence isn't hard-coded
  • For different queries, the LLM generates completely different plans
  • Use structured formats (JSON or XML) for reliable parsing
// JSON Plan Example
[
{
"step": 1,
"tool": "getItemDescriptions",
"arguments": { "shape": "round" }
},
{ "step": 2, ... },
]
Code as Action

Code Is the Ultimate Plan

Creating narrow tools for every query is brittle. Let the LLM write code.

Narrow Tools (Brittle)

getColumnMax, filterRows, getColumnMean, sumRows... doesn't cover all queries.

Code as Action (Flexible)

LLM writes Python with pandas. Access to hundreds of library functions it already knows.

Research (Wang et al.) confirms code-as-action outperforms tool-only approaches across multiple models.
Multi-Agent

Multi-Agent Workflows

Like hiring a team rather than a single person.

Researcher

Market Analyst

Analyzes trends, researches competitors. Tool: web search.

Designer

Graphic Designer

Creates visualizations, generates images. Tools: image APIs, code.

Writer

Copywriter

Transforms research into text and marketing copy. No special tools.

Workflows: Linear (Researcher → Designer → Writer) · Hierarchical (Manager coordinates specialists) · All-to-All (experimental collaboration)
Patterns

Communication Patterns

Three patterns for multi-agent systems.

01

Linear

A → B → C. Most common. Each agent passes output to the next.

02

Hierarchical

Manager coordinates specialists. Can extend to deeper hierarchies.

03

All-to-All

Experimental. Any agent can talk to any other. Unpredictable but creative.
